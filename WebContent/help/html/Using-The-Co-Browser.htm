Using The Co-Browser

Co-browsing is the feature of OPA that allows two or more reviewers to simultaneously conduct a review.

Steps:

To start co-browsing, start the co-browse servlet (Click on the link provided on the parent page). On the main screen you create a session (channel, presentation). Just type the name for new session. Have all persons you want to co-browse with you to start the servlet also. As soon as you do that, everybody who is running the same servlet will be able to join this session (channel). They need to choose a session from the list of opened sessions and type their own nickname (for real-time chat).

As a session owner you will see a list of users joined to your channel. Type the OPA URL  for Tracking and Review , such as(http://www.yoursite.com:8080/admim?fuseaction=review)  in the control frame. All your users will see the same. The browser on the client side does not need to be refreshed. Your client will see the same page entered in your window without his doing anything! Hit any link on the requested page or type another URL and again your client will see it!

Co-browse supports real-time chat. So you may discuss items on the proposal as you review them with participants.
